syn_encoding Attribute

The syn_encoding attribute specifies the encoding style for an FSM in the design. This attribute takes effect only when the FSM Compiler is enabled and cannot be set at a global level.

ValueDescription
default Automatically picks an encoding style that results in optimum performance for the FSM.
onehot Implements the FSM using a single flip-flop per state. Only two bits of the state register change, and only one of the bits is hot. The value determines the state.
gray Implements the FSM using gray encoding. Only one of the state registers changes at a time, and all the state registers are hot. This value determines the state.
sequential Implements the FSM with an unlimited number of bits that can change at one time. all the state registers are hot. This value determines the state.
safe Implements the FSM with default encoding. If the state machine reaches an invalid state, reset logic is added to force the state machine to a known state.
original Implements the FSM in the user-specified style and performs reachability analysis.

For detailed information about this attribute, use the Help button in the attribute Wizard, or the online help in the synthesis tool.
